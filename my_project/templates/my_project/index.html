Distributed Network and TCP Port Scanner

A distributed network and port scanner consists of the following components:

  • A Master node on which a web front-end runs. This is used to schedule scans and store the scan results.
  • Scan Agents that receive scan jobs from the Master node and conduct the actual network and port scan.

The Master node serves the Web UI using Django web framework and Bootstrap. The scan agents subscribe to this Master node when they are ready to accept scan requests. The Master then distributes the scan requests among the multiple scan agents which are subscribed.

The following functionalities are supported:

  • Check whether an IP address is alive
  • Find which IP address are alive in a block of IP addresses
  • Port scan an IP address for a given set of ports using any of the following scanning modes:
    • Full TCP Connect Scan
    • TCP SYN Scan
    • TCP FIN Scan
  • Scan ports sequentially or in random order
